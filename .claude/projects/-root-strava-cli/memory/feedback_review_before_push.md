description: Always review all changed code thoroughly before committing/pushing
type: feedback
---

Review all changed files carefully before committing and pushing. Don't rush to push.

**Why:** User wants clean, correct code pushed — not "fix it in the next commit" iterations.

**How to apply:** After making changes, read back all modified files, verify build, check for unused imports, dead code, inconsistencies, before committing.
